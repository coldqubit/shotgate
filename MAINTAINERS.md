# Maintainers

shotgate is an independent open-source project developed under the
[coldqubit](https://github.com/coldqubit) project home. This file lists who maintains it
today and how the roster changes. How decisions are made lives in
[GOVERNANCE.md](GOVERNANCE.md).

## Current maintainers

| Maintainer | Role | Areas |
| --- | --- | --- |
| [coldqubit](https://github.com/coldqubit) | Lead maintainer | All areas |

There is one maintainer today. That is stated plainly on purpose: the project is built
to take on more (see [Becoming a maintainer](#becoming-a-maintainer)), not built around a
single person. Governance, the Architecture Decision Records in [docs/adr/](docs/adr/),
and the container-first reproducibility exist so that the project does not depend on any
one person remaining available.

Reach the maintainers through [GitHub issues](https://github.com/coldqubit/shotgate/issues),
or at `info@coldqubit.org` for private matters. Security reports follow
[SECURITY.md](SECURITY.md).

## Becoming a maintainer

Co-maintainers are welcome, including for a single area such as a backend, the Terraform
module, or the docs. There is no expectation that a maintainer owns the whole project.
The path:

1. Land a few well-scoped pull requests (see [CONTRIBUTING.md](CONTRIBUTING.md)).
2. Help triage issues and review other contributors' pull requests for a while.
3. An existing maintainer proposes you in a pull request that adds you to this file and
   to the `@coldqubit/maintainers` team. Per [GOVERNANCE.md](GOVERNANCE.md), lazy
   consensus of the current maintainers confirms it.

New maintainers are added to the `@coldqubit/maintainers` team so that
[CODEOWNERS](CODEOWNERS) routes reviews to the role rather than to an individual.

## Emeritus

None yet. A maintainer who steps back is moved here with thanks, keeping credit for past
work and dropping only the review obligation.
